Tags

AUX has many tags.

#Info Tags

#id

Read-OnlyVideo

The ID of the bot.

Every bot has an automatically generated unique ID that cannot be changed.

#space

Read-Only

The space the bot lives in.

Every bot is stored in a space that specifies how long the bot lives and if it is shared with other people.

#Possible values are:

ValueDescription
shared
The bot is saved on the AUXPlayer and is shared with everyone else. (default)
local
The bot is saved on the device it was created from and is not shared with other devices.
tempLocal
The bot is not saved and is not shared with other sessions. When the current session is closed (by refreshing or closing the tab), the bot is destroyed.
history
The bot represents a mark (save point). The history space is not available by default. You need to use the server.browseHistory() function to load it. Additionally, the history space is read-only. This means that bots cannot be added or removed from it and they also cannot be edited.

#auxCreator

Video

The ID of the bot that created this bot.

#Behavior Tags

#auxDraggable

Video

Whether the bot is able to be dragged.

#Possible values are:

ValueDescription
true
The bot is able to be dragged. (default)
false
The bot cannot be dragged.

#auxDraggableMode

The mode that controls how a bot can be dragged.

#Possible values are:

ValueDescription
all
The bot can be dragged anywhere. (Default)
none
The bot cannot be dragged.
moveOnly
The bot can be moved anywhere within the dimension it is currently in but not moved to another dimension. (only applies to AUXPlayer)
pickupOnly
The bot can be moved to another dimension but not within the dimension it is currently in. (only applies to AUXPlayer)

#auxPositioningMode

The positioning mode that the bot uses.

#Possible values are:

ValueDescription
stack
The bot can be stacked upon other bots and other bots can be stacked upon it. (default)
absolute
The bot will ignore all other bots for positioning.

#auxDestroyable

Whether the bot is able to be destroyed in AUXPlayer. The bot can still be destroyed in Channel Designer.

#Possible values are:

ValueDescription
true
The bot will allow destruction events to occur outside Channel Designer. (default)
false
The bot will not allow destruction events to occur to it outside of Channel Designer.

#auxEditable

Whether the bot is able to be edited in the sheet in Channel Designer.

Use with caution. Once you set #auxEditable to false, you can't edit the bot via the sheet.

#Possible values are:

ValueDescription
true
The bot can be edited via the sheet. (default)
false
The bot cannot be edited via the sheet. (including being able to set auxEditable back to true)

#auxListening

Determines whether this bot responds to whispers or shouts. If this tag is set to true it will respond to them. If set to false, it will not.

#Possible values are:

ValueDescription
true
The bot will respond to whispers and shouts. (default)
false
The bot will not respond to whispers or shouts.

#auxConfigBot

The ID of the bot that "configures" this bot. When set, the config variable will be available in scripts and formulas as a shortcut to the specified bot.

#auxScale

Video

The uniform scale of the bot.

Defaults to 1.

#auxScaleX

Video

The X scale (width) of the bot. Combines with the #auxScale tag to determine the final width of the bot.

Defaults to 1.

#auxScaleY

Video

The Y scale (length) of the bot. Combines with the #auxScale tag to determine the final length of the bot.

Defaults to 1.

#auxScaleZ

Video

Defaults to 1.

#auxForm

The shape that the bot should be displayed as.

#Possible values are:

ValueDescription
cube
Cube mesh (default)
sphere
Sphere mesh
sprite
Camera facing (billboarded) sprite/quad images.
mesh
A mesh loaded from the web. See ##auxFormSubtype for additional options.

#auxFormSubtype

The subtype that the form should use. Useful for specifying how a mesh should be loaded.

#Possible values are:

ValueDescription
null
No subtype (default)
gltf
A GLTF mesh loaded from a URL.

#auxFormAddress

The address that the bot should represent data from.

When #auxForm is set to cube, sphere, or sprite, the address should be the URL of the image that the bot should display.

When #auxForm is set to mesh and #auxFormSubtype is set to gltf, the address should be the URL of the GLTF file that should be displayed.

#auxGLTFVersion

The GLTF specification version that should be used to load a GLTF model.

#Possible values are:

ValueDescription
2
Load the GLTF model with version 2 of the specification. (default)
Numbers < 2
Load the GLTF Model with version 1 of the specification. This is useful for loading meshes from poly.google.com.

#auxProgressBar

Causes a progress bar to be shown for the bot. Range is between 0 and 1. Needs both auxProgressBarBackgroundColor and auxProgressBarColor to visualize the progress

#Possible values are:

ValueDescription
Numbers >= 0 and <= 1
  • 0 means that you will only see #auxProgressBarBackgroundColor.
  • 0.5 means that you will see half of #auxProgressBarBackgroundColor and half of #auxProgressBarColor.
  • 1 means that you will only see #auxProgressBarColor.

#auxProgressBarColor

The foreground color of the progress bar.

#Possible values are:

ValueDescription
black
(default)
Any X11 ColorX11 colors are a list of standard colors that web browsers support.
Any Hex ColorHex colors are three or six digit numbers that specify a color by its red, green, and blue components.

#auxProgressBarBackgroundColor

The background color of the progress bar.

ValueDescription
white
(default)
Any X11 ColorX11 colors are a list of standard colors that web browsers support.
Any Hex ColorHex colors are three or six digit numbers that specify a color by its red, green, and blue components.

#auxProgressBarPosition

The anchor position for the progress bar.

#Possible values are:

ValueDescription
top
Top of the bot facing world oriented up. (default)
front
Front of the bot facing world oriented forward.
back
Back of the bot facing world oriented back.
right
Right of the bot facing world oriented right.
left
Left of the bot facing world oriented left.
floating
Floating above the bot.

#Dimension Tags

#[dimension]

Add a bot to a dimension.

Example: Set #room1 to true for the bot to appear in a dimension named room1.

#Possible values are:

ValueDescription
false
Removes the bot from the dimension. (default)
true
Puts the bot into the dimension.

#[dimension]SortOrder

The order that the bot should be in a stack of bots.

When bots are stacked, they are placed from the bottom to the top based on the sort order. So a bot with a sort order of 1 is going to be placed on top of a bot with a sort order of 2.

#Possible values are:

ValueDescription
Any NumberSpecifies the sort order of a bot in a stack of bots. (Default is 0)

#[dimension]X

The X (east-west) position of a bot on the grid in the specified dimension.

Integer values are mapped directly to the grid. e.g. 1.5 means halfway between the 1 and 2 grid squares.

#Possible values are:

ValueDescription
Any NumberSpecifies the X position of a bot in a dimension. (Default is 0)

#[dimension]Y

The Y (north-south) position of a bot on the grid in the specified dimension.

Integer values are mapped directly to the grid. e.g. 1.5 means halfway between the 1 and 2 grid squares.

#Possible values are:

ValueDescription
Any NumberSpecifies the Y position of a bot in a dimension. (Default is 0)

#[dimension]Z

The Z (up-down) offset that the bot is offset above/below the grid.

#Possible values are:

ValueDescription
Any NumberSpecifies the Z position of a bot in a dimension. (Default is 0)

#[dimension]RotationX

The X (pitch) rotation in radians that the bot should be rotated by.

#Possible values are:

ValueDescription
Any NumberSpecifies the X rotation of a bot in a dimension. (Default is 0)

#[dimension]RotationY

The Y (roll) rotation in radians that the bot should be rotated by.

#Possible values are:

ValueDescription
Any NumberSpecifies the Y rotation of a bot in a dimension. (Default is 0)

#[dimension]RotationZ

The Z (yaw) rotation in radians that the bot should be rotated by.

#Possible values are:

ValueDescription
Any NumberSpecifies the Z rotation of a bot in a dimension. (Default is 0)

#History Tags

History tags are tags that are automatically applied to bots in the history space.

#auxHistory

History Bot

Whether the bot is in the auxHistory dimension.

#auxMarkHash

History Bot

The SHA-256 hash that the mark represents.

#auxPreviousMarkHash

History Bot

The #auxMarkHash of the mark that was created before this mark. You can use this tag as a way to find previous marks from a mark.

#auxMarkTime

History Bot

The time that the mark was created.

#Player Bot Tags

All tags on the player bot will be synced with the URL query parameters. This means that setting the #abc tag on the player bot to hello will update the URL to include abc=hello. It also means that putting auxInventoryPortal=myDimension into the query string will set the #auxInventoryPortal tag to myDimension.

#auxUniverse

Player Bot

The universe that is loaded into auxPlayer.

#auxPagePortal

Player Bot

The dimension that is loaded into the "page" portal in auxPlayer. Only available on the player bot.

#auxSheetPortal

Player Bot

The dimension that is loaded into the "sheet" portal in auxPlayer. Only available on the player bot.

#auxInventoryPortal

Player Bot

The dimension that is loaded into the "inventory" portal in auxPlayer. Only available on the player bot.

#auxMenuPortal

Player Bot

The dimension that is loaded into the "menu" portal in auxPlayer. Only available on the player bot.

#Hidden Tags

Hidden tags are tags that are hidden in the sheet by default.

You can make your own hidden tags by adding a `_` (underscore) to the start of the tag.

#_auxEditingBot

Player Bot

The ID of the bot that the user is currently editing. Only available on the player bot.
